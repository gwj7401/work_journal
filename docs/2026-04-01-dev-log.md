# 开发日志 - 2026-04-01

## 今日任务细节 (What was done)

### 1. 集成 Physical Grid V3 表格提取方案
- **改进逻辑**：根据 `test_physical_grid_v3.py` 的测试结果，将主程序 `pdf2word_ocr.py` 的表格处理逻辑彻底整合。
- **高采样率渲染**：将 PDF 渲染的 Matrix 缩放到 **3.3x**。经测试，这是捕捉扫描件中极小字符（如序号 1-9）的临界点。
- **物理聚类算法 (Y-Clustering)**：
    - 废弃了基于 `RapidLayout` 边界的局限性处理，改为全局 Y 轴坐标聚类。
    - 聚类阈值设为行高的 **0.7 倍**（实验证明这对重叠度较高的扫描表格最稳健）。
- **3 列网格映射 (X-Mapping)**：
    - 针对当前业务场景，基于图片宽度的 **17%** 和 **46%** 位置设置逻辑分界线。
    - 强制按照 1:6:3 的列宽比例还原 Word 表格，确保名称列（中间列）有足够的空间展示多行文字。
- **样式深度紧实化**：
    - 全面采用 `Pt(10)` (十点字) 与 `仿宋_GB2312`。
    - 行距锁定为 `1.0`，前后段落间距调整为 `0`，彻底消除表格内不必要的留白。

### 2. 核心代码库优化
- 同步更新了 `pdf2word_ocr.py` 中的渲染与引擎调用逻辑，移除了之前版本中可能导致 Broadcast Error 的不完整参数设置。

## 下步计划
